Docker

What is Docker?

Docker is a containerization platform that allows you to run applications in isolated environments called containers. Containers are lightweight and portable, and they provide a consistent and reliable way to deploy applications.

Dockerfile

  • FROM: defines the base image use to start the build process
  • ADD: Copies the files from a source on the host into the container own filesystem at the set destination and as well as urls also added but copy cant do that
  • COPY: same as ADD
  • CMD: can be used for executing a specific command with in the container
  • ENTRYPOINT: set a default application to be used every time a container is created with the image
  • ENV: set environment variables
  • EXPOSE: set the specific port to enable networking between container and to the outside
  • MAINRAINER: defines full name and email address of image creater
  • RUN: it is central executive directory for Docker files
  • USER: set the UID which is to run the containert
  • VOLUME: used to enable access from the container to a dirctory on the host machine
  • WORKDIR: set the path where the command, defines the CMD is to be executed
  • LABEL: allows you to add a label to your docker image

Kubernetes

Introduction to Kubernetes

Kubernetes is a container orchestration platform due to managing more no of container it is risk and difficult to manage the docker swarm and kubernetes are the container orchestration tools. It is developed and maintained by Google. It does not provide CI/CD.

Kubernetes Capabilities

  • Automated rollouts and rollbacks
  • Storage orchestration
  • Horizontal scaling
  • Automated bin packing
  • Secret and configuration management
  • IPv4/IPv6 addresses
  • Batch execution
  • Self-healing
  • Service discovery and load balancing

Kubernetes Architecture

Cluster: A group of machines (nodes) working together to run containerized applications.

  • Control Plane (Brain): Manages the desired state of the cluster and makes decisions.
  • Worker Nodes (Machines): Run the actual applications (containers).

Kubernetes Objects

Kubernetes Objects: Persistent entities with identity, state, and behavior (like real-world objects).

  • Pods: The smallest deployable unit, containing one or more containers.
  • Namespaces: Isolate groups of resources within a cluster (e.g., by team, project).

Kubernetes Deployment

A Deployment is a way to manage the rollout of new versions of an application.

  • kubectl create deployment nginx-deployment --image=nginx:1.14.2
  • kubectl get deployments
  • kubectl describe deployment nginx-deployment
  • kubectl delete deployment nginx-deployment

Kubernetes Service

A Service is an abstraction over a set of Pods that defines a network interface and a set of endpoint policies.

  • kubectl expose deployment nginx-deployment --type=LoadBalancer --port=80
  • kubectl get svc
  • kubectl describe svc nginx-deployment
  • kubectl delete svc nginx-deployment

Kubernetes Persistent Volumes

A Persistent Volume (PV) is a piece of networked storage that has been provisioned by an administrator.

  • kubectl create pv pv-name --capacity=5Gi --access-modes=RWX --storage-class=local-storage
  • kubectl get pv
  • kubectl describe pv pv-name
  • kubectl delete pv pv-name

Kubernetes StatefulSets

A StatefulSet is a way to manage stateful applications, such as databases.

  • kubectl create statefulset mysql --image=mysql:5.7 --replicas=3
  • kubectl get statefulsets
  • kubectl describe statefulset mysql
  • kubectl delete statefulset mysql
